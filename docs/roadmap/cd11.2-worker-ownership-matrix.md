# CD11.2 Worker Ownership Matrix

## Purpose

This document is the repo-local ownership reference for launching W17 and W18 Workers after the W16 reset.

It defines package ownership, package-root reservations, package README reservations, global docs reservations, release classifier serialization, static test ownership, acceptance-file ownership, CI/root script serialization, parked-overlay rules, and likely fan-in triggers.

This document does not implement source behavior, does not modify tests or CI, and does not claim adapter readiness.

## Active target

The active target is:

```text
OpenClaw/Telegram adapter ready for real-system integration
```

This means fake E2E coverage, readiness aggregation, durable adapter-state evidence, no-leak coverage, package-root no-side-effect evidence, and opt-in secret-gated real-edge evidence can support a final release classification.

It is not production readiness. It does not include production listener/webhook/polling runtime, production credential loading, production durable backend, deployment runtime, sidecar runtime, OCA runtime expansion, or LifeOS/domain behavior.

## Universal Worker prompt fields

Every W17/W18 prompt should include these fields:

```text
SLICE:
PARALLEL_WAVE:
PARALLEL_GROUP:
SLICE_SHAPE:
OWNED_FILES_DIRECTORIES:
ALLOWED_SHARED_FILES:
FORBIDDEN_FILES_DIRECTORIES:
PACKAGE_ROOT_POLICY:
DOCS_POLICY:
TESTS_POLICY:
ACCEPTANCE_FILE_POLICY:
STATIC_FILE_POLICY:
FAN_IN_EXPECTATION:
PARKED_OVERLAYS:
READINESS_CLAIM_LIMIT:
```

A prompt is incomplete if it allows a package root, global docs, root scripts, workflows, package metadata, release classifier docs, broad static files, or acceptance files without naming the exact path and reservation.

## Reservation vocabulary

### Package root policy

| Value | Meaning |
|---|---|
| `none` | no package-root edit is allowed |
| `local-barrel-only` | local barrels under owned directories are allowed; package root remains untouched |
| `reserved` | one package root and one export slot are reserved by the prompt |
| `deferred` | implementation may land without root export; a later root-export slice may expose it |
| `serialized` | package root must be edited by only one Worker after sibling conflicts are resolved |

`reserved` must name:

```text
reserved_package_root:
reserved_export_slot:
```

Default: `none`.

### Docs policy

| Value | Meaning |
|---|---|
| `none` | no docs edit is allowed |
| `package-readme-reserved` | one package README or named section is reserved |
| `global-docs-reserved` | one global doc or named section is reserved |
| `release-classifier-reserved` | release classifier docs are reserved by one release-closure Worker |

Default for implementation lanes: `none`.

### Acceptance file policy

| Value | Meaning |
|---|---|
| `none` | no acceptance test edit is allowed |
| `unique-file` | Worker owns exactly one acceptance file |
| `serialized` | Worker owns a shared acceptance or acceptance-summary file after parallel leaves land |

Default for W17H1-W17H6: `unique-file` with the exact file listed below.

### Static file policy

| Value | Meaning |
|---|---|
| `none` | no static test edit is allowed |
| `new-file-only` | Worker may add one new static file named by prompt |
| `broad-static-reserved` | Worker may edit broad static topology/boundary files |
| `release-gate-static` | Worker owns static updates for release closure |

Default for implementation lanes: `none` unless local static tests are explicitly assigned.

## Wave 2 ownership — W17 implementation lanes

Wave 2 implements isolated vertical lanes. It must not add production runtime, default network calls, provider SDK/client construction in safe roots, direct `process.env` reads in libraries, OCA behavior, or LifeOS behavior.

### W17A — Plugin Runtime Composition

| Field | Value |
|---|---|
| Slice shape | vertical-integration |
| Primary package | `@hazeteam/openclaw-plugin-runtime` |
| Owned directories | `packages/openclaw-plugin-runtime/**` as narrowed by prompt |
| Package root | `packages/openclaw-plugin-runtime/src/index.ts` only if reserved |
| Package README | `packages/openclaw-plugin-runtime/README.md` only if reserved |
| Tests | package-local unit/static tests if assigned |
| Claim limit | local contract/fake-readiness evidence only |

Must preserve side-effect-free runtime composition, safe registry snapshots, no provider SDK imports, no network, and no secrets.

### W17B1-W17B4 — Adapter foundation lanes

| Slice | Owned concern | Primary package | Package root default |
|---|---|---|---|
| W17B1 | inbound mapping and command intent composition | `@hazeteam/openclaw-adapter` | deferred/no edit |
| W17B2 | host facade and command facade composition | `@hazeteam/openclaw-adapter` | deferred/no edit |
| W17B3 | rendering and delivery composition | `@hazeteam/openclaw-adapter` | deferred/no edit |
| W17B4 | callback, permission, approval composition | `@hazeteam/openclaw-adapter` | deferred/no edit |

Default rule: W17B lanes must use owned directories and local barrels. `packages/openclaw-adapter/src/index.ts` may be edited by only one reserved Worker or by a later package-root fan-in.

Adapter lane invariants:

- no private `hazeteam-core` imports;
- no copied core source;
- no raw provider payloads in public DTOs;
- no raw callback payloads in public DTOs;
- provider acknowledgement is not business success;
- permission is checked before token consume;
- no OCA/LifeOS dependency.

### W17C-W17F1 — Telegram transport lanes

| Slice | Owned concern | Primary package | Package root default |
|---|---|---|---|
| W17C | provider-shaped update quarantine, safe channel event, topic route | `@hazeteam/openclaw-telegram-transport` | deferred/no edit |
| W17D | outbound delivery request, injected provider port, ack/business split | `@hazeteam/openclaw-telegram-transport` | deferred/no edit |
| W17E | callback descriptor, permission, verify, consume, safe response | `@hazeteam/openclaw-telegram-transport` | deferred/no edit |
| W17F1 | config/readiness projection and credential-reference readiness | `@hazeteam/openclaw-telegram-transport` | deferred/no edit |

Default rule: W17C-W17F1 use local barrels and owned directories. `packages/openclaw-telegram-transport/src/index.ts` may be edited by only one reserved Worker or by a later transport package-root fan-in.

Transport lane invariants:

- raw provider input is quarantined and not surfaced publicly;
- topic routing authority is `channelRef + chatRef + threadRef`, not topic title;
- delivery provider ack remains distinct from business completion;
- callback flows preserve permission-before-token-consume;
- library code does not read `process.env`;
- missing credentials or provider ports produce safe blocked readiness, not fake pass.

### W17G1-W17G4 — Storage and testkit lanes

| Slice | Owned concern | Primary packages | Package root default |
|---|---|---|---|
| W17G1 | topic binding records/fakes | adapter/testkit | deferred/no edit |
| W17G2 | callback token records/fakes | adapter/testkit | deferred/no edit |
| W17G3 | delivery attempt records/fakes | adapter/testkit | deferred/no edit |
| W17G4 | inbound idempotency records/fakes | adapter/testkit | deferred/no edit |

Default rule: W17G lanes may add deterministic fakes and safe records only in assigned paths. Production durable backends remain future work. Adapter/testkit package roots require explicit reservation.

Storage/testkit lane invariants:

- no raw provider payload storage;
- no token/secret/credential values in public records;
- replay does not duplicate token consumption, delivery, runtime dispatch, or approval state;
- fake readiness is not production readiness.

## Wave 2.5 ownership — W17H acceptance matrix

W17H Workers are acceptance-slice Workers. Each owns exactly one file and one concern.

| Slice | Unique owned file | Required proof |
|---|---|---|
| W17H1 | `tests/acceptance/w17h1-inbound-fake-e2e.test.mjs` | provider-shaped update to safe event to topic route to command intent |
| W17H2 | `tests/acceptance/w17h2-outbound-fake-e2e.test.mjs` | adapter result to render model to delivery request to provider ack/business split |
| W17H3 | `tests/acceptance/w17h3-callback-permission-fake-e2e.test.mjs` | callback to permission to token verify to token consume to safe response |
| W17H4 | `tests/acceptance/w17h4-durable-replay-fake-e2e.test.mjs` | inbound, callback, and delivery replay safety |
| W17H5 | `tests/acceptance/w17h5-no-leak-matrix.test.mjs` | public outputs JSON-serializable and free of raw provider data, tokens, secrets, paths, stack traces, provider handles |
| W17H6 | `tests/acceptance/w17h6-package-root-no-side-effect.test.mjs` | root imports create no provider client, network, env read, durable connection, listener, timer, delivery, callback execution, or token consume |

Rules:

- do not append to a shared acceptance file during parallel W17H;
- do not change implementation behavior unless the prompt explicitly assigns it;
- do not add real network calls;
- do not require secrets;
- do not use OCA or LifeOS as adapter readiness evidence;
- W17H may claim at most local fake-E2E evidence, not final adapter readiness.

## Wave 3 ownership — W18 real-edge preparation

Wave 3 prepares explicit edges. It must not turn default imports, tests, or checks into production runtime.

| Slice | Shape | Owned concern | Serialization notes |
|---|---|---|---|
| W18A | vertical-integration | runtime value boundary, credential resolver interfaces, redacted readiness | W18B must not depend on unmerged W18A code unless serialized or explicitly assigned |
| W18B | vertical-integration | provider client port boundary, redacted provider result, ack/business split | no provider SDK/client construction in safe roots |
| W18D | contract/interface | listener, webhook, polling interfaces or docs | no daemon, listener startup, webhook server, or polling loop unless explicitly assigned |
| W18F1 | docs | runtime edge docs | no release classifier claim |

Wave 3 may introduce interfaces and opt-in boundaries only when assigned. Default `npm run check` must remain no-network and secret-free.

## Wave 4 ownership — release closure

Wave 4 classifies evidence. It does not imply production readiness.

| Slice | Shape | Owned concern | Serialization notes |
|---|---|---|---|
| W18C | vertical-integration | secret-gated real-smoke local files/tests | run after W18A/W18B; smoke must be opt-in, redacted, and status-precise |
| W18E1 | release-closure | release gate static/CI/root script evidence | owns root scripts/workflows/package metadata only if explicitly reserved |
| W18E2 | release-closure | release docs/current-state/known limitations/package READMEs | owns release classifier docs only after evidence is known |
| W18E3 | release-closure | final adapter readiness report | last; final classifier only |

Release closure status vocabulary:

```text
not-ready
contract-ready
fake-e2e-ready
secret-gated-ready
adapter-ready-for-real-system-integration
production-ready
```

Only W18E3 may make the final `adapter-ready-for-real-system-integration` claim. No W18 slice may claim `production-ready` unless a separate production-candidate scope implements and tests production runtime, credential loading, durable backends, deployment runtime, operational health/readiness, and recovery behavior.

## Package README reservations

Package READMEs are shared docs. They require exact file reservation or named-section reservation.

| README | Reservation rule | Claim limit |
|---|---|---|
| `packages/openclaw-plugin-runtime/README.md` | one owner: W17A for package-local update or W18E2 for final docs | no final adapter claim |
| `packages/openclaw-adapter/README.md` | one owner: serialized adapter docs slice or W18E2 | no final adapter claim before W18E3 |
| `packages/openclaw-telegram-transport/README.md` | one owner: W17C/D/E/F1 section reservation, W18F1, or W18E2 | skipped/blocked smoke not pass |
| `packages/openclaw-testkit/README.md` | one owner: W17G/H section reservation or W18E2 | no production readiness claim |
| `packages/oca-wrapper/README.md` | parked-overlay docs only | cannot be adapter readiness evidence |
| `packages/domain-lifeos/README.md` | parked-overlay docs only | cannot be adapter readiness evidence |

No Worker may update package README release claims from leaf-local evidence alone.

Global release claims must serialize. If W18E2 and W18E3 both need a release classifier surface, W18E3 runs last.

## Static ownership

Static tests should guard permanent architecture and safety invariants:

- public core imports only;
- no copied core source;
- no provider SDK/client construction in safe roots;
- no default network or secret requirement;
- no raw provider/callback payloads in public outputs;
- no token/secret/credential/path/stack/provider-handle leaks;
- package-root imports have no side effects;
- provider acknowledgement is not business success;
- permission before token consume;
- OCA/LifeOS parked overlays are not adapter readiness evidence.

Static tests should not guard temporary phase topology, temporary absence/presence of files, branch names, Worker bookkeeping, or stale historical W8/W9/W14/W15 assumptions unless those are still real invariants.

Broad global static edits serialize under a dedicated static cleanup or W18E1 release-gate slice.

## CI, scripts, package metadata, and lockfile serialization

These surfaces are high fan-in and must not be touched by parallel implementation leaves unless explicitly assigned:

- root `package.json` scripts;
- package `package.json` metadata;
- `package-lock.json`;
- `.github/workflows/**`;
- `scripts/**`;
- release gate script definitions;
- default `npm run check` composition.

Rules:

- default `npm run check` must remain no-network and no-secret;
- secret-gated smoke must remain opt-in and separately classified;
- adding acceptance checks to default gates is a release-gate decision and should serialize;
- package metadata changes must serialize with lockfile updates if a lockfile change is required.

## Parked-overlay rules

OCA and LifeOS remain parked downstream until the adapter-ready-for-real-system-integration gate is actually proven and classified.

Forbidden before that gate:

- OCA runtime behavior;
- real OCA client execution;
- OCA credential loading;
- production OCA runtime;
- LifeOS/domain product behavior;
- product-specific agent catalogs;
- product-specific command catalogs;
- sidecar runtime;
- production deployment runtime;
- using OCA or LifeOS fixtures as adapter readiness proof.

Allowed only with explicit assignment:

- compile/test fixes that keep the repository green;
- no-leak or security fixes;
- docs clarifying parked downstream status;
- static checks preventing accidental parked-overlay expansion.

## Likely fan-in triggers

No fan-in is required by this W16E slice. Future fan-in or serialization is likely if any of these conditions occur:

| Trigger | Preferred response |
|---|---|
| multiple W17B lanes need `packages/openclaw-adapter/src/index.ts` | local barrels first; later adapter package-root fan-in if needed |
| multiple W17C-W17F1 lanes need `packages/openclaw-telegram-transport/src/index.ts` | local barrels first; later transport package-root fan-in if needed |
| adapter/testkit storage lanes need both roots | reserve one root/export slot at a time or fan in after leaves |
| several Workers need one package README | serialize by named sections or defer to W18E2 |
| root scripts/workflows/default `check` need changes | W18E1 or explicit tooling fan-in |
| broad static topology cleanup is required | dedicated static cleanup or W18E1 |
| release checklist/current-state/limitations need final evidence | W18E2/W18E3 serialized release closure |
| acceptance evidence needs a summary | W18E3 report or explicit acceptance-summary slice |

## Worker stop rule

If a Worker needs an unreserved shared file, it must not silently edit it. The report should include:

```text
BLOCKED_SHARED_FILE:
needed_file:
reason:
safe_partial_work_completed:
recommended_followup:
```

Safe owned-file work may still be completed only if the prompt allows partial completion and the unreserved file is not required for correctness of the partial result.
